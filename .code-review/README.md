## 审查流程

1. 全量阅读项目源代码
2. 分析架构设计和代码质量
3. 识别 Bug、安全隐患和改进点
4. 按优先级分类问题 (高/中/低)
5. 生成审查报告并归档

## 问题优先级说明

- 🔴 **高优先级** - 影响功能正确性或安全性的 Bug，需立即修复
- 🟡 **中优先级** - 影响可维护性或用户体验，建议近期修复
- 🟢 **低优先级** - 优化建议，可在迭代中逐步完善
